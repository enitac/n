📚 Portal User Manual

1. Namitete Secondary Portal (Front Page)

The front page is designed for students, parents, guardians and sponsors to access exam results quickly.

How to View Results:

  • On the home page
  • Click on: Get Started button.
  • Choose portal:-Exam Day or Exam ODL.
  • Open portal of your choice: by clicking buttons Exam Day or Exam ODL.
  • Select Form: Choose the student's class (Form 1–4).
  • Enter Exam Number: Enter the unique ID (e.g., J047 or 004)
  • Enter Password: (Default is — unless disabled by Admin).
  • View Results: Click "View Results" for the official Progress Report or the secondary button for Brief Results.

Features of the Progress Report:

  • A4 Optimized: Designed to fit perfectly on one page when printed.
  • Automated Comments: The system generates teacher and head teacher remarks based on the student's average.
  • Sharing: Use the 📱 WhatsApp button to send a PDF directly to a parent's phone, or click Save (PDF) to keep a digital copy.

2. Administrator Login

To access the backend, staff must follow these steps:

  • On the main page, click LOGIN.
  • System Password: Enter _ _ _ _.
  • Code: Enter your Code. (This is required to save any changes. Without this, you can only view data, not save it).

3. Data Entry (For Teachers)

This section is used to record student scores for specific subjects.

  • Navigate to Admin Dashboard > Data Entry.
  • Select Class and Subject: Once both are selected, the student list will load automatically.
  • Enter Scores: Type the numeric score in the box next to the student's name.
  • The system uses Live Update: Scores are updated in the marksheet immediately as you type.
  • Special Indicators: This applies to forms 3 and 4 - Enter N for No Mark, Enter X for Not Enrolled.
  • SAVE DATA: Crucial! After entering marks, you must click Save All Data to permanently store the scores on the server.
  • Real-Time Monitor: In the Admin Home, use the Subject Score Entry Monitor to see which subjects have been completed and which are still missing scores.

4. File Uploads & Configuration

This area is for the Exam Coordinator or Headteacher.

A. Managing Student Lists (CSV)

  • Upload CSV: Select a class and upload a .csv file.
  • Format: The file must have headers: STUDENT'S NAME, Exam Number,SEX,SNE.
  • Manual Entry: If a single student is missing, use the Manual Student Entry section at the bottom of the Uploads page to add them without re-uploading the whole list.

B. Report Settings

  • Update the Term, Academic Year, Next Term Opening Date, and School Fees. These details will appear automatically on every student's report.

C. Teacher Assignment

  • Select a class and enter the names of the teachers for each subject. This ensures the correct teacher's name appears next to the subject grade on the Progress Report.

D. Grade Boundary Editor

  • Forms 1 & 2: Edit the minimum scores for grades A, B, C, D, and F.
  • Forms 3 & 4: Edit the minimum scores for the 1–9 point system.
  • Note: Changing these values will instantly update all student grades and the pass/fail analysis.

5. Sheets & Analysis (Reporting)

Access this via Admin Dashboard > Progress, Grade, Analysis Sheets.

  • Mark Sheet: Displays raw scores for all students in a grid.
  • Grade Sheet: Displays the converted letter/number grades and calculates Class Position and Rank.
  • All these sheet have an option of Saving files as Excel or Pdf, the sheets can be printed as final sheets for display on the notice board.
  • Analysis Report: Provides a subject-by-subject breakdown (number of A's, B's, etc.) and the Pass/Fail percentage for the entire class.
  • Batch PDF Generator: Click Progress Report Generator (Batch PDF). This powerful tool will automatically generate and download individual PDF reports for every student in the selected class simultaneously.

6. Access Control (Security)

In the Uploads page, the administrator can:

  • Lock/Unlock Subjects: to prevent teachers from changing scores after a deadline.
  • Portal Access Control: Toggle the password requirement on or off.
  • Recommendation: Disable access (set to Disabled) until all marks are verified to prevent students from seeing incomplete results.

⚙️ Technical Troubleshooting

  • Data not saving? Ensure your "Code" is entered correctly in the Admin Home.
  • Table looks messy on mobile? The portal is "Surgical Fixed" for mobile. Tables are scrollable horizontally. For the best experience, use "Landscape" mode on your phone.
  • Printing issues? When printing sheets (Mark Sheet/Grade Sheet), the system automatically formats them in Landscape. When printing Progress Reports, it formats them in Portrait. Don't change these settings manually in the print dialog.
